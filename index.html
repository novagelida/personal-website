Prolog treasure finder

Prolog based agent able to find one or more treasures inside a generic scenario without any a-priori knowledge. The scenario is composed by tiles and each of these tiles could represent different kind of terrain like mountain or river. The agent, using some heuristics and the A* algorithm, explore the map until he can see a treasure. When the agent is able to see a treasure (different tiles allow different seeing conditions) he makes a plan to cath it.

  • Exam: Artificial Intelligence
  • Date: January 2013 - April 2013
  • Team Size: 3
  • My role: Code design and programming
  • Techs: swi-prolog

Orientation within a virtual environment

The goal of the project is to put the user into a urban virtual environment within which he has to follow a randomly generated path from point A to point B. The performance will be evaluated using metrics that serve to establish if the subject is able to orientate within the scenario. The application is designed to be used inside the virtual theater of the University of Milan. In order to immerse users in the virtual by letting them interact naturally with their body and to have the most vivid perceptual experience, the scene is presented using stereoscopy and the interactions are realized with MS Kinect.

  • Exam: Immersive and interactive virtual environment
  • Date: November 2012 - October 2013
  • Team Size: 3
  • My role: Code design, UI programming, Businnes logic
  • Techs: python, panda3D, MS kinect, Vrpn, FAAST, OpenNI

Colorful Island

What are cookies?

Cookies are small text files containing a string of characters that can be placed on your computer or mobile device that uniquely identify your browser or device.

Cookies allow a site or services to know if your computer or device has visited that site or service before. cookies can then be used to help understand how the site or service is being used, help you navigate between pages efficiently, help remember your preferences, and generally improve your browsing experience.
